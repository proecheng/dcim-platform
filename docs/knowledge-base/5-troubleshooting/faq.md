## 1. 登录与权限

<details>
<summary><strong>Q: 忘记登录密码怎么办？</strong></summary>

**A:** 请联系系统管理员重置密码。

管理员操作路径: 系统管理 → 用户管理 → 找到用户 → 重置密码

默认重置密码: `Dcim@123`（首次登录强制修改）
</details>

<details>
<summary><strong>Q: 为什么登录后看不到某些菜单？</strong></summary>

**A:** 菜单可见性由角色权限控制。

| 角色 | 可见模块 |
|------|----------|
| 运维人员 | 监控、告警、巡检 |
| 能源管理员 | 监控、告警、能源、节能 |
| 管理员 | 全部模块 |

如需扩展权限，请联系管理员。
</details>

<details>
<summary><strong>Q: 登录提示 "Token过期"？</strong></summary>

**A:** Token有效期为24小时，过期后需重新登录。

如频繁出现此问题，请检查:
1. 浏览器时间是否正确
2. 是否在多个标签页同时登录
3. 清除浏览器缓存后重试
</details>

## 2. 监控与告警

<details>
<summary><strong>Q: 数据显示 "---" 或 "NaN" 是什么意思？</strong></summary>

**A:** 表示数据采集失败或设备离线。

**排查步骤:**
1. 检查设备通信状态（设备管理页面）
2. 查看点位配置是否正确
3. 确认数据采集服务是否运行

**临时处理:** 页面右上角点击"刷新"重新获取数据
</details>

<details>
<summary><strong>Q: 为什么收到很多重复告警？</strong></summary>

**A:** 可能是告警阈值设置过于敏感，导致数值在阈值附近波动时反复触发。

**解决方案:**
1. 调整告警阈值，增加死区范围
2. 启用告警屏蔽（设置时间窗口内相同告警只触发一次）
3. 检查传感器是否异常

**操作路径:** 告警管理 → 告警规则 → 编辑规则 → 设置死区
</details>

<details>
<summary><strong>Q: 告警声音太吵，如何关闭？</strong></summary>

**A:** 可以按级别设置声音提醒。

**操作路径:** 个人设置 → 告警偏好
- 1级告警: 建议保留声音（紧急）
- 2-4级告警: 可选择关闭

> ⚠️ 不建议关闭1级告警声音，可能错过重要事件
</details>

<details>
<summary><strong>Q: 如何快速查找历史告警？</strong></summary>

**A:** 使用告警历史页面的高级筛选功能。

**筛选条件:**
- 时间范围
- 告警级别
- 设备/点位
- 处理状态
- 关键词搜索

**操作路径:** 告警管理 → 历史告警 → 高级筛选
</details>

## 3. 能源管理

<details>
<summary><strong>Q: PUE数据不准确，和实际计算有偏差？</strong></summary>

**A:** PUE计算依赖准确的总能耗和IT能耗数据。

**排查步骤:**
1. 检查总能耗计量点是否配置正确
2. 检查IT负载计量点是否遗漏
3. 对比电表读数验证数据准确性

**常见原因:**
- UPS输入功率未计入
- 照明/办公区能耗错误计入
- 计量表倍率配置错误

**操作路径:** 能源管理 → 计量配置 → 检查点位映射
</details>

<details>
<summary><strong>Q: 如何设置峰谷电价时段？</strong></summary>

**A:** 在电价配置页面设置。

**操作路径:** 能源管理 → 电价配置

**常见时段划分 (以工业电价为例):**
| 时段 | 时间 | 电价 |
|------|------|------|
| 尖峰 | 10:00-12:00, 18:00-21:00 | 1.40元 |
| 高峰 | 08:00-10:00, 12:00-18:00, 21:00-23:00 | 1.00元 |
| 平段 | 07:00-08:00, 23:00-24:00 | 0.65元 |
| 低谷 | 00:00-07:00 | 0.35元 |

> 注意: 具体电价请以当地电力公司公告为准
</details>

<details>
<summary><strong>Q: 需量预警是什么意思？</strong></summary>

**A:** 最大需量即将超过合同容量，可能产生超容罚款。

**应对措施:**
1. 立即检查哪些设备负载较高
2. 启动负荷转移（如有配置）
3. 手动关闭非关键设备
4. 联系能源管理员确认

**预警阈值:** 默认为合同容量的 90%
</details>

## 4. 节能方案

<details>
<summary><strong>Q: 点击"智能分析"没有生成任何方案？</strong></summary>

**A:** 可能原因:

1. **数据不足:** 需要至少30天历史数据
2. **能耗特征不明显:** 没有明显的优化空间
3. **设备配置不完整:** 缺少关键设备信息

**解决方案:**
- 检查 `energy_hourly` 表是否有足够数据
- 完善 `power_devices` 设备配置
- 配置 `device_shift_configs` 转移参数
</details>

<details>
<summary><strong>Q: 方案预期收益和实际收益差距很大？</strong></summary>

**A:** 常见原因及解决方案:

| 原因 | 解决方案 |
|------|----------|
| 基础数据不准确 | 查看数据追溯，核实输入数据 |
| 执行条件变化 | 参考RL优化建议调整参数 |
| 设备响应不及时 | 检查控制系统延迟 |
| 外部因素影响 | 考虑气温、负载变化等 |

**操作路径:** 方案详情 → 效果监测 → 查看达成率分析
</details>

<details>
<summary><strong>Q: 措施执行失败怎么办？</strong></summary>

**A:** 查看执行日志确定失败原因。

**常见原因:**
- 设备通信中断
- 权限不足
- 设备处于保护状态
- 目标值超出设备范围

**操作路径:** 方案详情 → 措施列表 → 查看执行日志
</details>

<details>
<summary><strong>Q: RL优化建议是什么？要不要采纳？</strong></summary>

**A:** RL(强化学习)会基于执行效果给出参数微调建议。

**建议采纳场景:**
- 达成率低于80%
- 环境条件发生变化
- 需要进一步优化

**不建议采纳场景:**
- 达成率已超过90%
- 建议与安全规范冲突
- 探索率高(初期学习阶段)

**查看探索率:** 方案详情 → RL状态 → exploration_rate
- < 0.1: 稳定期，建议可信度高
- > 0.2: 探索期，建议仅供参考
</details>

## 5. 数据与报表

<details>
<summary><strong>Q: 如何导出历史数据？</strong></summary>

**A:** 多个页面支持数据导出。

**支持导出的页面:**
- 能源报表 → 导出Excel
- 告警历史 → 导出Excel
- 设备台账 → 导出Excel

**导出格式:** Excel (.xlsx)

**数据量限制:** 单次最多导出10000条记录
</details>

<details>
<summary><strong>Q: 报表数据和实时数据不一致？</strong></summary>

**A:** 报表数据基于历史归档，可能有以下时延:

| 报表类型 | 更新频率 | 数据截止时间 |
|----------|----------|--------------|
| 日报 | 每天 02:00 | 前一天 24:00 |
| 周报 | 每周一 02:00 | 前一周周日 24:00 |
| 月报 | 每月1日 02:00 | 上月最后一天 24:00 |

如需最新数据，请查看实时监控页面。
</details>

<details>
<summary><strong>Q: 数据字典在哪里查看？</strong></summary>

**A:** 参考知识库文档:

- [数据字典](../4-data-guides/data-dictionary.md)
- [计算公式大全](../4-data-guides/formula-reference.md)

**或通过API文档:** http://localhost:8080/docs
</details>

## 6. 系统配置

<details>
<summary><strong>Q: 系统响应很慢怎么办？</strong></summary>

**A:** 可能原因及解决方案:

1. **网络问题:** 检查网络连接
2. **数据量过大:** 减少页面查询范围
3. **浏览器缓存:** 清除缓存后刷新
4. **服务器负载:** 联系管理员检查

**自检步骤:**
1. 按F12打开开发者工具
2. 查看Network标签，检查请求耗时
3. 如API响应>3秒，联系技术支持
</details>

<details>
<summary><strong>Q: 如何添加新设备？</strong></summary>

**A:** 需要管理员权限。

**操作路径:** 资产管理 → 设备台账 → 新增设备

**必填信息:**
- 设备编码
- 设备名称
- 设备类型
- 额定功率
- 所属区域

添加后还需配置:
1. 关联监测点位
2. 配置转移参数 (如需参与负荷转移)
</details>

<details>
<summary><strong>Q: 支持移动端访问吗？</strong></summary>

**A:** 系统支持响应式设计，可在移动端浏览器访问。

**推荐浏览器:**
- iOS: Safari
- Android: Chrome

**功能限制:**
- 大屏3D展示在移动端性能受限
- 建议使用PC端进行配置操作
</details>
