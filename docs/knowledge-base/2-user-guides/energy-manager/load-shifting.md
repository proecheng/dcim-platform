# 负荷转移操作指南

> 峰谷电价套利的完整配置和操作流程

---

## 概述

负荷转移是指将高电价时段（峰/尖峰）的用电负荷转移到低电价时段（谷/深谷），从而降低电费支出。

### 适用场景
- 执行峰谷电价的用户
- 有可调度负荷的数据中心
- 峰谷电价差 > 0.3元/kWh

### 节能潜力
- 年节省: 10-50万元（取决于可转移负荷量）
- 典型峰谷价差: 0.6-1.0元/kWh

---

## 操作流程

### 步骤1: 进入负荷转移页面

**菜单路径**: 能源管理 → 负荷转移

页面包含四个区域:
1. **统计卡片** - 可转移功率、年度潜在收益
2. **设备列表** - 可转移设备及其配置
3. **方案规划器** - 拖拽式转移规则配置
4. **效益汇总** - 预计节省金额

### 步骤2: 查看可转移设备

系统自动识别可转移设备:

| 设备类型 | 默认可转移比例 | 说明 |
|----------|---------------|------|
| 水泵 (PUMP) | 40% | 可变频调速 |
| 空调 (AC) | 30% | 有舒适度约束 |
| 暖通 (HVAC) | 35% | 柔性中等 |
| 照明 (LIGHTING) | 50% | 可分区调光 |
| 冷水机 (CHILLER) | 25% | 有温度约束 |
| UPS | 0% | ❌ 不可转移 |
| IT服务器 | 0% | ❌ 不可转移 |

### 步骤3: 调整设备配置

点击设备卡片的「配置」按钮:

```yaml
可配置项:
  is_shiftable: true           # 是否可转移
  shiftable_power_ratio: 0.4   # 可转移比例 (0-1)
  allowed_shift_hours:         # 允许转移时段
    - 0, 1, 2, 3, 4, 5, 6      # 凌晨谷时
  forbidden_shift_hours:       # 禁止转移时段
    - 9, 10, 11, 14, 15, 16    # 工作高峰
  min_continuous_runtime: 2    # 最小连续运行时长 (小时)
  requires_manual_approval: true  # 是否需要人工审批
```

### 步骤4: 创建转移规则

在「方案规划器」中创建转移规则:

1. 点击「添加规则」
2. 选择源时段（如：峰时 10:00-12:00）
3. 选择目标时段（如：谷时 02:00-06:00）
4. 选择参与设备
5. 设置转移功率和时长

**规则示例**:
```
规则1: 峰→谷 转移
├── 源时段: 峰时 (10:00-12:00)
├── 目标时段: 谷时 (02:00-06:00)
├── 设备: 冷冻水泵1#, 冷冻水泵2#
├── 转移功率: 60kW
└── 转移时长: 4小时
```

### 步骤5: 查看收益预估

系统实时计算预期收益:

```
日节省计算:
├── 转移功率: 60kW
├── 转移时长: 4小时
├── 源时段电价: 1.0元/kWh (峰时)
├── 目标时段电价: 0.35元/kWh (谷时)
├── 电价差: 0.65元/kWh
└── 日节省: 60 × 4 × 0.65 = 156元

年度收益:
└── 156元 × 250工作日 = 39,000元 ≈ 3.9万元
```

### 步骤6: 提交方案

1. 确认所有规则配置正确
2. 点击「提交方案」
3. 进入审批流程（如需要）
4. 审批通过后自动执行

## 可转移比例计算

### 系统默认值

基于设备类型的经验值，见上表。

### 基于历史数据的推荐值

系统可根据历史数据计算更精准的可转移比例:

```
推荐比例 = min(
    1 - (历史最低功率 / 额定功率),     # 最低功率约束
    (最大功率 - 平均功率) / 额定功率,  # 波动空间
    峰时用电占比 × 柔性系数,           # 峰时可转移比例
    设备类型上限                       # 类型约束
) × 安全系数(0.85)
```

**查看推荐值**: 设备卡片 → 「查看推荐值」

## 约束条件设置

### 时间约束

```yaml
allowed_shift_hours:   # 允许转移的时段 (0-23)
  - [0, 1, 2, 3, 4, 5, 6]  # 凌晨

forbidden_shift_hours: # 禁止转移的时段
  - [9, 10, 11]            # 上午高峰
  - [14, 15, 16]           # 下午高峰
```

### 运行约束

```yaml
min_continuous_runtime: 2.0  # 最小连续运行时长 (小时)
max_shift_duration: 4.0      # 最大转移持续时间 (小时)
min_power: 20.0              # 最低运行功率 (kW)
max_ramp_rate: 5.0           # 最大功率变化率 (kW/分钟)
```

### 审批约束

```yaml
shift_notice_time: 30         # 转移通知提前时间 (分钟)
requires_manual_approval: true  # 是否需要人工审批
```

## 常见问题

<details>
<summary><strong>Q: 为什么某些设备不在可转移列表中？</strong></summary>

**A:** 设备需满足以下条件:
1. `is_enabled = true` (设备已启用)
2. 设备类型非关键负荷 (非UPS/IT_SERVER)
3. 已配置 `device_shift_configs` 记录

缺少配置的设备可联系管理员添加。
</details>

<details>
<summary><strong>Q: 转移规则冲突怎么办？</strong></summary>

**A:** 系统会自动检测冲突:
- 同一设备同时段多个规则 → 优先级高的生效
- 源时段和目标时段重叠 → 规则无效

建议在规划器中检查规则时间线视图。
</details>

<details>
<summary><strong>Q: 实际转移量和预期不符？</strong></summary>

**A:** 常见原因:
1. 设备实际可调容量小于配置值
2. 目标时段负载已满
3. 安全约束触发限制
4. 通信延迟导致未及时响应

查看执行日志获取详细原因。
</details>

## 最佳实践

1. **从小规模开始**: 先配置1-2台设备，验证效果后再扩展
2. **设置合理约束**: 预留安全裕量，避免影响核心业务
3. **关注季节变化**: 夏季制冷需求高，可转移空间小
4. **定期检视效果**: 每月回顾转移效果，调整策略
