# 节能方案操作指南

> 从智能分析到执行监测的完整操作流程

---

## 概述

节能方案功能是DCIM系统的核心模块，基于深度学习算法自动分析数据中心能耗特征，生成可执行的节能措施方案。

### 功能入口
- **菜单路径**: 能源管理 → 节能中心 → 智能建议
- **快捷访问**: http://localhost:3000/energy/suggestions

---

## 操作流程

### 步骤1: 触发智能分析

1. 进入「智能建议」页面
2. 点击 **「智能分析」** 按钮
3. 系统将自动：
   - 分析过去30天的能耗数据
   - 识别可优化的设备和时段
   - 基于6种模板生成节能方案

```
分析周期: 过去30天
分析维度: 峰谷用电、设备负载、温度分布
输出结果: 1-6个节能方案（根据数据特征）
```

### 步骤2: 查看方案列表

分析完成后，方案以卡片形式展示：

| 字段 | 说明 |
|------|------|
| 方案编号 | 格式: `A1-20260201-001` |
| 方案名称 | 如「空调群控优化方案」|
| 预期收益 | 年度节能金额（万元）|
| 优先级 | 高/中/低 |
| 状态 | 待处理/已接受/执行中/已完成 |

### 步骤3: 查看方案详情

点击方案卡片进入详情页，可查看：

#### 3.1 基本信息
- 方案类型和描述
- 分析时间范围
- 生成时间

#### 3.2 措施列表
每个方案包含多个具体措施：

```
措施示例:
├── M1: 空调1#温度设定调整 (当前26℃ → 目标27℃)
├── M2: 冷水机2#变频优化 (当前50Hz → 目标45Hz)
└── M3: 新风机组启停优化 (高峰时段关闭)
```

#### 3.3 数据追溯 (专利S3)
展开任意措施可查看完整计算链路：

```
可转移功率 = 额定功率 × 可转移比例
         = 100kW × 0.35
         = 35kW

数据来源:
├── 额定功率: power_devices.rated_power (设备ID: 12)
└── 可转移比例: device_shift_configs.shiftable_power_ratio (默认值0.35)
```

#### 3.4 ML预测信息 (专利S2)
- **置信度**: 模型对预测结果的信心程度
- **特征重要性**: 影响预测的关键因素
- **预测区间**: 节能收益的上下限

### 步骤4: 接受方案

1. 勾选要实施的措施（可部分选择）
2. 点击 **「接受方案」** 按钮
3. 确认选择后，方案状态变为「已接受」

> ⚠️ **注意**: 接受后系统会重新计算预期收益（仅计算选中措施）

### 步骤5: 执行方案

1. 点击 **「执行」** 按钮
2. 系统将：
   - 采集各措施的基准值 (S4a)
   - 下发控制指令（模拟/实际）
   - 启动持续监测 (S4b)

执行结果反馈：
```json
{
  "executed_count": 3,      // 执行措施数
  "success_count": 3,       // 成功数
  "monitoring_started": true // 监测已启动
}
```

### 步骤6: 效果监测

执行后，系统自动进入监测阶段：

#### 6.1 实时监测数据
- 当前功率 vs 基准功率
- 累计节能量 (kWh)
- 累计节省金额 (元)

#### 6.2 效果达成率 (S4d)
```
效果达成率 = 实际节能收益 ÷ 预期节能收益 × 100%
```

#### 6.3 RL优化建议 (S5)
系统会基于监测数据给出参数调整建议：
- 温度设定微调
- 设备启停时间优化
- 功率分配重新平衡

## 方案模板说明

| 模板ID | 名称 | 适用场景 | 典型收益 |
|--------|------|----------|----------|
| A1 | 空调温度优化 | 夏季制冷高峰 | 5-15万/年 |
| A2 | 照明智能控制 | 非核心区域 | 2-5万/年 |
| A3 | UPS效率优化 | 低负载运行 | 3-8万/年 |
| A4 | 冷却系统联动 | 冷机+水泵 | 20-50万/年 |
| A5 | 新风系统优化 | 过渡季节 | 5-10万/年 |
| B1 | 峰谷负荷转移 | 峰谷电价差 | 10-30万/年 |

## 常见问题

### Q1: 分析没有生成任何方案？
**原因**: 数据量不足或能耗特征不明显
**解决**:
- 确保有30天以上的历史数据
- 检查设备配置是否完整

### Q2: 预期收益和实际收益差距大？
**原因**: 基础数据不准确或执行条件变化
**解决**:
- 查看数据追溯检查输入数据
- 参考RL优化建议调整参数

### Q3: 措施执行失败？
**原因**: 设备通信异常或权限不足
**解决**:
- 检查设备连接状态
- 联系管理员确认控制权限
